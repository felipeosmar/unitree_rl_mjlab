Decisões de design

  • Onde fica o microfone?
    • Laptop que comanda → mais simples, latência de rede irrelevante
    • Cabeça do Go2 → ruído do gait, mas robô fica autônomo
  • Wake-word? "Go2..." via openWakeWord evita STT rodar 100% do tempo
  • VAD (Voice Activity Detection): corta silêncios e segmenta enunciados — economiza ~10× tempo de inferência

FASE 6 · CONTROLE DE TRAJETÓRIA

Waypoints → comando de velocidade

Pure Pursuit

Mira sempre num lookahead point à frente do robô na trajetória.

Math (simplificado)

def pure_pursuit(pose, path, Ld=0.6):
    # encontra ponto na trajetória a Ld de distância
    target = lookahead(pose, path, Ld)
    dx = target.x - pose.x
    dy = target.y - pose.y
    # transforma pro frame do robô
    alpha = atan2(dy, dx) - pose.theta

    vx = V_NOMINAL * cos(alpha)
    wz = 2 * V_NOMINAL * sin(alpha) / Ld
    return (vx, 0.0, wz)  # vy=0, holonômico opcional

Tuning

  • Ld grande → suave mas corta curvas
  • Ld pequeno → preciso mas oscila
  • Saturar vx ≤ 2 m/s, |ωz| ≤ 1 rad/s — limites do treino

O ponto de injeção

Em State_RLBase.cpp hoje o command vem do gamepad. Pra integrar a navegação autônoma, basta trocar a fonte de (vx, vy, ωz).

  • Joystick continua como override de segurança
  • RT+A entra em modo Velocity → policy autônoma
  • LT+B sempre volta pra Passive (kill switch)

Sem retreinar a policy

A policy já aceita qualquer (vx, vy, ωz) dentro do range do treino:

  • vx ∈ [-1.0, 2.0] m/s
  • ωz ∈ [-1.0, 1.0] rad/s

Pure-pursuit precisa saturar dentro disso, senão o robô extrapola e cai (ver AUMENTAR_VELOCIDADE_GO2.md).
